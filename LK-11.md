### Bug 1: Duplikasi Rute dan Konflik Nama Rute (Conflicting Routes)
* **Lokasi File**: [routes/web.php](file:///d:/laragon/www/lk10/routes/web.php)
* **Deskripsi Masalah**:
  Rute `/login` dan `/logout` didefinisikan beberapa kali. Rute `/login` pertama mengarah ke `AuthController@login` dengan nama rute `login`, namun ditimpa oleh definisi rute `/login` kedua yang hanya memanggil *closure view* dengan nama rute `login.page`. Akibatnya, nama rute default `login` hilang, yang dapat memicu `RouteNotFoundException` saat middleware `auth` melakukan *redirect* otomatis. Selain itu, rute `/logout` kedua menggunakan *closure* yang memanggil kelas facade `Auth` tanpa mengimpor *namespace*-nya.
* **Sebelum Perbaikan**:
  ```php
  Route::get('/', function () {
      return view('login'); 
  })->name('login.page');

  Route::get('/login', [AuthController::class, 'login'])->name('login');
  Route::get('/auth/callback', [AuthController::class, 'callback'])->name('auth.callback');
  Route::post('/logout', [AuthController::class, 'logout'])->name('logout');

  Route::get('/login', function () {
      return view('login');
  })->name('login.page');

  Route::get('/login/workos', [AuthController::class, 'login'])
      ->name('login.workos');

  Route::post('/logout', function () {
      Auth::logout();
      return redirect('/login');
  })->name('logout');
  ```
* **Sesudah Perbaikan**:
  ```php
  // Redirect root ke halaman login
  Route::get('/', function () {
      return redirect()->route('login');
  });

  // Render halaman login (menyediakan rute bernama 'login' untuk redirect auth otomatis)
  Route::get('/login', function () {
      return view('login');
  })->name('login');

  // Rute inisiasi login OAuth ke WorkOS
  Route::get('/login/workos', [AuthController::class, 'login'])->name('login.workos');

  // Callback WorkOS
  Route::get('/auth/callback', [AuthController::class, 'callback'])->name('auth.callback');

  // Logout terpusat melalui AuthController
  Route::post('/logout', [AuthController::class, 'logout'])->name('logout');
  ```
* **Penjelasan & Dampak**:
  Rute dibersihkan dari duplikasi. Rute `/login` kini secara konsisten mengembalikan halaman login dan memiliki nama rute `login` sehingga sistem autentikasi bawaan Laravel dapat melakukan *redirect* ke halaman login dengan lancar saat mengakses rute terproteksi. Rute `/logout` juga disatukan ke `AuthController@logout` untuk menghindari kode yang tidak rapi.

### Bug 2: Penggunaan Fungsi `env()` Secara Langsung di Controller
* **Lokasi File**: [app/Http/Controllers/AuthController.php](file:///d:/laragon/www/lk10/app/Http/Controllers/AuthController.php)
* **Deskripsi Masalah**:
  Controller `AuthController` memanggil fungsi pembantu `env()` secara langsung untuk mengambil data kredensial WorkOS (`WORKOS_CLIENT_ID`, `WORKOS_API_KEY`, dan `WORKOS_REDIRECT_URI`). Di Laravel, ketika perintah *config cache* dijalankan di server produksi (`php artisan config:cache`), file `.env` tidak lagi dibaca dan fungsi `env()` akan mengembalikan nilai `null`. Hal ini dapat menyebabkan fitur integrasi login WorkOS rusak total di lingkungan produksi.
* **Sebelum Perbaikan**:
  ```php
  // Di AuthController.php -> login()
  $clientId = env('WORKOS_CLIENT_ID');
  $redirectUri = env('WORKOS_REDIRECT_URI');

  // Di AuthController.php -> callback()
  $response = Http::post(
      'https://api.workos.com/user_management/authenticate',
      [
          'client_id'     => env('WORKOS_CLIENT_ID'),
          'client_secret' => env('WORKOS_API_KEY'),
          'code'          => $code,
          'grant_type'    => 'authorization_code',
      ]
  );
  ```
* **Sesudah Perbaikan**:
  *Menambahkan konfigurasi di [config/services.php](file:///d:/laragon/www/lk10/config/services.php):*
  ```php
  'workos' => [
      'client_id' => env('WORKOS_CLIENT_ID'),
      'api_key' => env('WORKOS_API_KEY'),
      'redirect_uri' => env('WORKOS_REDIRECT_URI'),
  ],
  ```
  *Mengakses nilai dari konfigurasi di `AuthController.php`:*
  ```php
  // Di AuthController.php -> login()
  $clientId = config('services.workos.client_id');
  $redirectUri = config('services.workos.redirect_uri');

  // Di AuthController.php -> callback()
  $response = Http::post(
      'https://api.workos.com/user_management/authenticate',
      [
          'client_id'     => config('services.workos.client_id'),
          'client_secret' => config('services.workos.api_key'),
          'code'          => $code,
          'grant_type'    => 'authorization_code',
      ]
  );
  ```
* **Penjelasan & Dampak**:
  Mengubah pengambilan nilai dari `env()` menjadi `config()`. Hal ini mematuhi standar *best practice* keamanan Laravel, memastikan bahwa konfigurasi tetap aman saat cache konfigurasi diaktifkan di server produksi.

### Bug 3: Rute Resource `show` Terbuka Tanpa Implementasi Metode di Controller
* **Lokasi File**: [routes/web.php](file:///d:/laragon/www/lk10/routes/web.php) & [app/Http/Controllers/BookController.php](file:///d:/laragon/www/lk10/app/Http/Controllers/BookController.php)
* **Deskripsi Masalah**:
  Kode rute menggunakan `Route::resource('books', BookController::class)`. Laravel secara otomatis menghasilkan rute detail untuk menampilkan item buku tunggal (`GET /books/{book}`, nama rute `books.show`) dan memetakan ke metode `show()` di `BookController`. Namun, metode `show()` tidak didefinisikan dalam `BookController`, sehingga pengguna atau crawler yang mengakses URL `/books/1` akan memicu error atau menampilkan halaman kosong yang tidak terduga.
* **Sebelum Perbaikan**:
  ```php
  // Di routes/web.php
  Route::middleware('auth')->group(function () {
      Route::resource('books', BookController::class);
  });
  ```
* **Sesudah Perbaikan**:
  ```php
  // Di routes/web.php
  Route::middleware('auth')->group(function () {
      Route::resource('books', BookController::class)->except(['show']);
  });
  ```
* **Penjelasan & Dampak**:
  Membatasi rute resource dengan mengecualikan (`except(['show'])`) tindakan yang tidak diimplementasikan. Jika pengguna memanggil URL `/books/{id}` dengan metode GET, Laravel kini akan secara otomatis merespons dengan status HTTP `405 Method Not Allowed` karena rute GET tersebut tidak terdaftar lagi (sedangkan PUT/DELETE untuk URI yang sama masih terdaftar). Hal ini mencegah akses tidak valid ke halaman kosong.

## 2. Refactoring Log (Log Refaktorisasi Kode)

Kami melakukan refaktorisasi pada 2 bagian kode di `BookController` untuk meningkatkan pemeliharaan kode (*maintainability*) dan menghilangkan duplikasi kode (*DRY - Don't Repeat Yourself*).

### Refaktorisasi 1: Ekstraksi Duplikasi Validasi ke Custom Form Request
* **File Baru**: [app/Http/Requests/BookRequest.php](file:///d:/laragon/www/lk10/app/Http/Requests/BookRequest.php)
* **Alasan Refaktorisasi**:
  Metode `store` (simpan) dan `update` (perbarui) di `BookController` mendefinisikan array aturan validasi yang hampir sama persis. Satu-satunya perbedaan adalah kolom `cover` yang bersifat wajib (`required`) saat pembuatan data buku baru dan bersifat opsional (`nullable`) saat pembaruan data buku. Duplikasi aturan validasi ini melanggar prinsip *DRY* dan menyulitkan pemeliharaan jangka panjang.
* **Sebelum Refaktorisasi**:
  ```php
  // Di BookController.php -> store()
  $request->validate([
      'title' => 'required|max:255',
      'author' => 'required|max:255',
      'publisher' => 'required|max:255',
      'year' => 'required|numeric',
      'category' => 'required|max:255',
      'description' => 'required',
      'cover' => 'required|image|mimes:jpg,jpeg,png|max:2048'
  ]);

  // Di BookController.php -> update()
  $request->validate([
      'title' => 'required|max:255',
      'author' => 'required|max:255',
      'publisher' => 'required|max:255',
      'year' => 'required|numeric',
      'category' => 'required|max:255',
      'description' => 'required',
      'cover' => 'nullable|image|mimes:jpg,jpeg,png|max:2048'
  ]);
  ```
* **Sesudah Refaktorisasi**:
  *Membuat kelas Form Request [BookRequest.php](file:///d:/laragon/www/lk10/app/Http/Requests/BookRequest.php):*
  ```php
  <?php

  namespace App\Http\Requests;

  use Illuminate\Foundation\Http\FormRequest;

  class BookRequest extends FormRequest
  {
      public function authorize(): bool
      {
          return true;
      }

      public function rules(): array
      {
          return [
              'title' => 'required|max:255',
              'author' => 'required|max:255',
              'publisher' => 'required|max:255',
              'year' => 'required|numeric',
              'category' => 'required|max:255',
              'description' => 'required',
              'cover' => $this->isMethod('POST') 
                  ? 'required|image|mimes:jpg,jpeg,png|max:2048'
                  : 'nullable|image|mimes:jpg,jpeg,png|max:2048'
          ];
      }
  }
  ```

### Refaktorisasi 2: Penyederhanaan Mass-Assignment Controller dengan Data Validasi
* **Lokasi File**: [app/Http/Controllers/BookController.php](file:///d:/laragon/www/lk10/app/Http/Controllers/BookController.php)
* **Alasan Refaktorisasi**:
  Setelah validasi berhasil, metode `store()` dan `update()` di `BookController` secara manual memetakan kembali satu demi satu data dari `$request` ke pemanggilan model (`Book::create` / `$book->update`). Hal ini tidak efisien karena kita mendefinisikan nama-nama kolom database secara berulang di controller. Dengan memanfaatkan Form Request, kita bisa langsung menggunakan `$request->validated()` untuk melakukan operasi database.
* **Sebelum Refaktorisasi**:
  ```php
  // Di BookController.php -> store()
  Book::create([
      'title' => $request->title,
      'author' => $request->author,
      'publisher' => $request->publisher,
      'year' => $request->year,
      'category' => $request->category,
      'description' => $request->description,
      'cover' => $cover
  ]);

  // Di BookController.php -> update()
  $book->update([
      'title' => $request->title,
      'author' => $request->author,
      'publisher' => $request->publisher,
      'year' => $request->year,
      'category' => $request->category,
      'description' => $request->description,
      'cover' => $cover
  ]);
  ```
* **Sesudah Refaktorisasi**:
  ```php
  // Di BookController.php -> store()
  public function store(BookRequest $request)
  {
      $validated = $request->validated();
      $validated['cover'] = $request->file('cover')->store('covers', 'public');

      Book::create($validated);

      return redirect()->route('books.index')
          ->with('success', 'Data buku berhasil ditambahkan');
  }

  // Di BookController.php -> update()
  public function update(BookRequest $request, Book $book)
  {
      $validated = $request->validated();

      if ($request->hasFile('cover')) {
          Storage::disk('public')->delete($book->cover);
          $validated['cover'] = $request->file('cover')->store('covers', 'public');
      }

      $book->update($validated);

      return redirect()->route('books.index')
          ->with('success', 'Data buku berhasil diupdate');
  }
  ```
